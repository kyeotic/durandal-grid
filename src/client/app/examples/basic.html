Basic - Simple Configuration

The following can be controlled via config properties passed to the grid.

  • Columns
    • header - The text of the HTML column header (can be left blank)
    • property - The property on the object to use as the data (can be left blank)
    • canSearch - boolean, if true the column will be searchable. Defaults to false.
    • canSort - boolean, if false the column will not be sortable. Defaults to true.
    • sort - a function that takes a left and right, and returns 1 or -1. Asc/Desc swapping will be handled by the plugin, you don't need to worry about it. Defaults to the javascript 'sort' function.
  • pageSize - How many rows will display per page
  • alwaysShowPaging - Boolean. If true, the paging controls will show even if there is only 1 page (defaults to false)
  • pageSizeOptions - An array of options to allow the user to control the page size (defaults to [25, 50, 75, 100])
  • showPageSizeOptions - Boolean. If true will show the buttons that allow the user to control the page size (defaults to false).

Example

Javascript

var data = [ /* Some data /* ];

return {
    gridConfig: { 
        data: data,
        pageSize: 5,
        showPageSizeOptions: true,
        pageSizeOptions: [5, 10, 15],
        columns: [
            { header: 'First Name', property: 'firstName' },
            { header: 'Last Name', property: 'lastName', canSearch : true },
            { header: 'Age', property: 'age', canSort: true, sort: function(a,b) { return a.age < b.age ? -1 : 1; } }
        ]
    }
};

HTML

<table class="paging-container grid-table" data-bind="grid: gridConfig"></table>
